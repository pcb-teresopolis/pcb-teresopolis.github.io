Nós trabalhadores desempenhamos um papel fundamental na estrutura social, sendo a espinha dorsal da economia e da produção da vida como é hoje.

A fim de partilhar socialmente a estrutura de manutenção e contribuição desse site, escrevi esse documento explicando alguns dos processos.

## Primeiro passo

O primeiro passo é criar uma conta no github, o processo é bem simples, com apenas 3 passos.

O github é uma plataforma online onde voce pode hospedar seu codigo, e é onde o codigo desse site está e também onde ele está sendo hopedado. Tudo de graça pela bondade da [microsoft dona do github desde 2018](https://g1.globo.com/economia/tecnologia/noticia/microsoft-compra-github-por-us-75-bilhoes.ghtml). Aprenda mais sobre o github na [comunidade do Brasil](https://github.com/github/brasil).

Para começar a usar o GitHub, será necessário criar uma conta pessoal gratuita no site deles e verificar o seu endereço de e-mail.

<ol>
  <li>Navegue até <a href="https://github.com/">https://github.com/</a>.</li>
  <li>Clique em <strong>Inscreva-se</strong>.</li>
  <li>Siga os prompts para criar sua conta pessoal.</li>
</ol>
Durante a inscrição, será solicitado que você verifique seu endereço de e-mail. Sem um endereço de e-mail verificado, você não poderá concluir algumas tarefas básicas no GitHub, como a criação de um repositório.

Segue o link para o repositório do codigo desse site: https://github.com/pcb-teresopolis/pcb-teresopolis.github.io

# Como contribuir de fato

Existem duas formas mais comuns de contribuir com a pagina, os problemas (ou issues no github em ingles), e as solicitações de mudança (ou pull requests no github em ingles).

## Problemas / Issues

É possivel encontrar a pagina de problemas clicando no botão do menu superior da pagina, como circulado estranhamento na foto a seguir.

![Screenshot](https://pcb-teresopolis.github.io/assets/img/github-problemas-issues.webp){: .mx-auto.d-block :}

Nessa parte é possivel acompanhar ou criar novas discussões, pedidos de ajuda, duvidas, debate de contúdo, sugestoes de melhoria, criticas etc

## Solicitações de alteração (Pull requests)

É possivel acessar a parte de pedidos de alteração do codigo / conteúdo da pagina clicando no botão do menu superior também como demonstrado na imagem a seguir.

![Screenshot](https://pcb-teresopolis.github.io/assets/img/github-pull-requests.webp){: .mx-auto.d-block :}

Nessa parte é possivel acompanhar ou criar solicitações de alteração ou inclusão de conteúdo. O conteúdo só pode ser alterado mediante aprovação dos nossos camaradas da celula que estão com seus pseudonimos mantendo esse repositório.

Cada post nesse site tem sua pagina gerada por um arquivo na linguagem [markdown (aprenda mais nesse post com exemplos de como usar markdown)](https://pcb-teresopolis.github.io/2024-05-17-exemplos-markdown/) ou também podem as vezes ser usando html.
Esse código usa uma ferramenta chamada Jekyll para ser traduzido nessa pagina do site que você esta lendo. Isso é feito atomaticamente no github assim que o codigo é alterado la.

Esse código então pode ser visto e editado pois fica armazenado nessa pagina onde está todo o codigo desse site aqui: https://github.com/pcb-teresopolis/pcb-teresopolis.github.io/tree/main/_posts

Dentro da pasta "_posts" temos todos os posts do site, por exemplo o post sobre markdown tem o caminho do arquivo "_posts/2024-05-17-exemplos-markdown.md". Exemplo na imagem a seguir dos dois posts que temos até o momento em que estou escrevendo esse texto.

![Screenshot](https://pcb-teresopolis.github.io/assets/img/github-posts.webp){: .mx-auto.d-block :}

Para editar esses arquivos e enviar a solicitação de mudança siga esse passo a passo do próprio github, o paragrafo de "Editar arquivos no repositório de outro usuário": https://docs.github.com/pt/repositories/working-with-files/managing-files/editing-files#editing-files-in-another-users-repository

E para enviar novos posts, ao criar um novo arquivo na pasta "_posts" tente duplicar um arquivo existente e editar em cima dele as configurações e conteúdo necessários.

Espero ter ajudado, qualquer dúvida abram n aparte de problemas do repositório: https://github.com/pcb-teresopolis/pcb-teresopolis.github.io/issues

Abraço, camaradas. venceremos.
